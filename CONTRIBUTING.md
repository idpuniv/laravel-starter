## Contributing

Thank you for considering contributing to this project. This project follows the contribution model of the Laravel framework.

### Bug Reports

To encourage active collaboration, we strongly encourage pull requests, not just bug reports. If you file a bug report, please include a clear description, relevant information, and a code sample to demonstrate the issue.

### Core Development Discussion

Proposals for new features or improvements can be discussed in the project's GitHub Discussions. For real-time informal discussion, you can reach out on our Discord channel (if you have one).

### Which Branch?

*   **Bug fixes** should be sent to the `main` branch.
*   **Minor features** that are fully backward compatible may be sent to the `main` branch.
*   **Major new features** or features with breaking changes should be sent to the `develop` branch.

### AI-Generated Contributions

We appreciate every contribution. However, contributions that are primarily AI-generated without thoughtful human review and consideration are not acceptable. If you use AI tools, the resulting code must be thoroughly reviewed, tested, and understood by you before submitting.

### Coding Style

This project follows the PSR-2/PSR-12 coding standards and uses a tool like StyleCI or PHP CS Fixer to ensure consistency. Don't worry if your code style isn't perfect, as our automated tools will handle it after the merge.

### Security Vulnerabilities

If you discover a security vulnerability, please email [your-email@example.com] instead of using the issue tracker.
